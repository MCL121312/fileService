# 变更检查清单（MVP）

每次让 Agent 改这个仓库时，优先按下面顺序思考：

1. 这是渐进式修改，还是会影响现有结构的大改？
2. 能否直接复用 `src/routes`、`src/core`、`src/templates` 现有能力？
3. 是否会影响现有接口返回结构？
4. 如果改了 API，是否需要同步 `src/openapi/spec.ts`？
5. 如果改了 dashboard，是否检查了 `public/app.js` 和 `public/styles.css`？
6. 如果改了任务/模板流，是否检查了 `taskManager` 与 `templateManager` 的职责边界？
7. 是否可以用最小范围验证，而不是跑高成本流程？

## 默认执行偏好

- 默认先读相关文件，再修改。
- 默认优先小步提交式思路，不一次性大面积改造。
- 默认保留现有中文注释风格与直观命名。
- 如果需求描述不完整，先补最小可用版本，再把可选扩展点单独列出。
