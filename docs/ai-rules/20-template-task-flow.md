# 模板与任务流规则（MVP）

## 现有职责边界

- `templateManager` 负责模板注册、读取和数据校验。
- `taskManager` 负责创建任务、排队处理、查询状态、删除任务、读取结果。
- 模板注册入口是 `src/templates/index.ts`，不要绕过它偷偷注入模板。

## 新能力接入方式

- 新的报告/文件生成能力，优先接入“模板 -> 任务 -> 文件”的现有链路。
- 不要直接在 route 中写重型生成逻辑，优先让 route 负责参数接收与响应组装。
- 不要在多个地方重复计算文件地址，优先复用现有 `getFileUrl` 风格。

## 状态与兼容性

- 现有任务状态以 `pending / processing / completed / failed` 为中心，新增逻辑时优先兼容这套状态机。
- 涉及文件可用性时，注意区分：
  - 任务存在但未完成
  - 任务完成但结果不可用
  - 文件扩展名与格式不匹配
- 如果要扩展任务信息，优先追加字段，不破坏已有读取逻辑。

## MVP 偏好

- 能挂在现有 manager 上就不要新建 manager。
- 能补现有模板注册流程就不要引入新插件体系。
- 先做单一路径跑通，再考虑通用抽象。
