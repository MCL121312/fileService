# 前端与 CSS 规则（MVP）

## 前端改动范围

- 当前前端以 `public/index.html`、`public/app.js`、`public/styles.css` 为主。
- 修改 dashboard 时优先做局部增强，不轻易改动整体结构和交互节奏。
- 如果只是补一个小功能，优先在现有页面与脚本上增量实现。

## CSS 规则

- 总是优先使用嵌套式 CSS 来组织样式。
- 在 CSS 中不要用 `&` 代替父级，除非是伪类/伪元素场景。
- 样式修改保持渐进式：
  - 先补局部 class
  - 再补局部样式块
  - 非必要不重排整份样式表
- 不要为了一个小改动引入新的 UI 框架或大规模改写 DOM 结构。

## 前后端联动

- 如果 dashboard 依赖任务接口字段，前后端修改要一起检查。
- 如果新增展示字段，优先保持旧字段继续可用，避免页面直接失效。

## 验证偏好

- 每次前端改动后，优先做最小验证：
  - 检查接口字段是否匹配
  - 检查页面是否还能正常加载
  - 检查交互是否没有明显报错
