版本 1.0.0

## 简介

原有项目使用word转pdf思路，word的样式操作比较复杂，面对复杂报告内容时不够灵活。所以需要一个可以用特定数据生成pdf文件的服务。

## 设计范围

本次设计聚焦“模板 -> 数据校验 ->PDF 生成 -> 任务管理 -> 文件输出”这条主链路。

## 重点解决的是

如何用结构化数据稳定生成 PDF 文件，并让生成过程具备可排队、可追踪、可查询结果的能力。

## 设计范围包括

- 模板组织方式与注册方式
- 模板数据校验规则
- HTML 模板到 PDF 的渲染链路
- 浏览器资源复用与并发控制任务创建、执行、状态查询、结果文件保存

## 设计范围不包括：

- Word 模板编辑与复杂样式操作体系
- 通用可视化模板编辑器
- 业务数据来源本身的采集与计算
- 多格式导出平台化能力
- 模板热更新、插件化管理等扩展能力
<!-- 纵向分层 -->

## 模块

- template 模块：负责模板的注册、读取、数据校验
- task 模块：负责任务的创建、执行、状态查询、结果文件保存
- pdfRenderer 模块：负责 HTML 模板到 PDF 的渲染
- browserPool 模块：负责浏览器资源的复用与并发控制
- database 模块：负责任务与日志的持久化
